# Homework 5: Terraform Infrastructure

Цей проєкт реалізує інфраструктуру AWS за допомогою Terraform, використовуючи модульний підхід та віддалений бекенд для зберігання стану, орієнтуючись на рекомендацію HashiCorp та AWS.

## Опис інфраструктури
Інфраструктура складається з трьох основних модулів:

- **s3_backend**: створює S3-бакет для зберігання стану Terraform та DynamoDB таблицю для блокування. Це забезпечує надійність та захист від конкурентних змін. Бакет має ввімкнене версіонування та політику ownership controls, що відповідає вимогам безпеки.
- **vpc**: створює VPC, три публічні та три приватні підмережі у Availability Zones. Для публічних підмереж налаштовано Internet Gateway, для приватних - NAT Gateway з Elastic IP. Також створюються маршрутні таблиці та асоціації, що забезпечують коректний доступ до інтернету.
- **ecr**: створює репозиторій Amazon Elastic Container Registry (ECR) з увімкненим скануванням образів при push. Політика доступу дозволяє базові операції push/pull для всіх користувачів, що відповідає типовим сценаріям CI/CD.

## Використання

1. Ініціалізація:
   ```bash
   terraform init
   ```

2. Преревірка конфігурації
   ```bash
   terraform validate
   ```

3. Перегляд плану
    ```bash
    terraform plan
    terraform plan -out="tfplan" (за бажанням)
    ```

4. Застосування
    ```bash
    terraform apply
    terraform apply "tfplan" (за бажанням)
    ```

5. Видалення
    ```bash
    terraform destroy
    ```

## Outputs

- `s3_bucket_name`: Назва S3-бакету для бекенду.
- dynamodb_table_name: таблиця для блокування.
- `vpc_id`: ID створеної VPC.
- `public_subnet`: Список ID публічних підмереж.
- `private_subnet`: Список ID приватних підмереж.
- `internet_gateway_id`: ідентифікатор IGW 
- `repository_url`:  URL ecr ркпозиторію


## Особливості

- Створення NAT Gateway займає кілька хвилин
- Перед видаленням ECR репозиторію потрібно очистити його від образів
- При використанні бекенду S3+DynamoDB Terraform може не видалити сам бекенд, оскільки він зберігає стан. Потрібна обов'язкова перевірка, на випадок необхідності видалення вручну
